# データ形式詳細説明

## 入力データ

### 登録学生一覧

* プログラムに登録されている学生の一覧

* 重要カラム (認定の計算に実際に使われるカラム)
  * **共通ID** UTokyo Account (10桁)
* それ以外にいくつカラムがあっても良い (アルゴリズムでは使われない)
* 推奨カラム (認定の計算に使われないが結果を目視, 確認しやすくするために推奨される)
  * 学籍番号
  * 学生氏名
* 例:
  
 |共通ID    |学籍番号  | 学生氏名   |
 |----------|----------|------------|
 |1234567890|88289817  |三村マサカズ|
 | 234567890|84952075  |大竹和樹    |
 |  ...     | ...      | ...        |

* 注
  * 同じ共通IDの行が複数あっても良い. 意味的には1つしかないのと同じ
    * 同じ共通IDに対して修士と博士で学籍番号が違うので自然に発生する
    * その場合, 出力においては, 一つのセルに複数の学籍番号が改行区切りで並ぶ
  * 全角数字があってもよい. 対応する半角文字と同じ
    * つまり `１２３４５６７８９０` は `1234567890` と同じ
    * つまり `12345６７８９０` も `1234567890` と同じ
  * 共通IDは10桁の数字だが, 10桁未満の場合は先頭に `0` が補われる
    * つまり上記 `234567890` は `0234567890` と同じ

### 科目一覧

* 重要カラム (認定の計算に実際に使われるカラム)
  * **科目コード**
  * **対象年度 (省略可)**
  * **組み合わせ科目群 (省略可)**
  * **排他科目群 (省略可)**
* それ以外にいくつカラムがあっても良い (出力の「全て」シートにに含まれる)

* 「科目コード」はUTASの「科目コード」と同じもの
* 「対象年度」は, ある科目コードの科目が, 特定の年度でだけ認定される場合に指定する. 例: `2022`, `2022-2024`, `2020, 2022, 2024` などの表記が可能. 何も書かなければ「毎年」の意味. カラムをまるごと省略すれば術の科目が毎年の意味.
* 「組み合わせ科目群」は, 複数の科目を全て単位取得して初めてそれらが認定に使われる, という場合に使われる. たとえば科目Aが1単位, 科目Bが1単位で A, Bを両方取ると両方の単位 (1+1=2単位) が認定に使われるような場合. そのような場合がなければ不要. 該当する科目群に対して「組み合わせ科目群」に同じ記号を書く. 例:

 |科目コード|組み合わせ科目群|
 |---------|----------------|
 |A        |1|
 |B        |1|
 |C        |2|
 |D        |2|
 |E        |2|

このように書くと, A, Bは両方取って初めて認められる, C, D, Eを全て取って初めて認められる, という意味になる
* 「排他科目群」は, 複数の科目群からいくつとってもそのうちの一つしか認定に使われない, という場合に使われる. 例えば 科目A, B, C (おそらく内容が似た科目) があり, それらの中からひとつしか認定には使わないような場合に使われる. 該当する科目群に対して「排他科目群」に同じ記号を書く. 例:

 |科目コード|排他科目群|
 |---------|----------------|
 |A        |1|
 |B        |1|
 |C        |2|
 |D        |2|
 |E        |2|

このように書くと, A, Bは両方取っても片方しか認定に使われない, C, D, Eから2つ以上とっても一つしか認定に使われない, という意味になる. なお認定に使われるのは, 取得した科目の中で単位数が最大のもの(おそらく殆どはどれも同じというケースであろうが). 例えば科目Aが2単位, 科目Bが3単位であれば, 3単位が認められる.

## 認定のアルゴリズム

* 登録学生一覧とUTASの「共通ID」を突き合わせて各学生が履修した科目, 年度, その合否を得る
* その結果と科目一覧の「科目コード」を尽きあわせて, 各学生が, 当該プログラムが認定対象としている科目, 年度で合格した科目を得る
* 各学生について, 組み合わせ科目群, 排他的科目群を考慮して, 認定に使う科目の集合を算出する
* 各学生について, 認定に使うこととなった科目の単位を合計(A)する
* それが一定数以上であれば認定(B)

## 出力データ

* `認定結果.xlsx` というExcelファイルに結果が書かれる
* 3つのシート「認定」「詳細」「全て」がある
  * 「認定」シートには各学生について認定された単位数の合計(上記の(A))およびそれが指定された必要単位数以上であるか(上記の(B))
  * 「詳細」シートにはその算出過程が書かれる. 各行に
    * 学生, 履修科目, 履修年度, 合否, 単位取得, 認定に使われるか否か, 認定に使われる単位数, などが書かれている
    *「詳細」シートの認定単位を各学生について合計したものが 「認定」シートだと思えば良い
  * 「全て」シートには「詳細」シートの内容に加え, 登録学生一覧や科目一覧に書かれていた情報が全て書かれる
